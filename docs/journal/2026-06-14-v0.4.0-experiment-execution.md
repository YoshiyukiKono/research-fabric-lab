## MinIO Connectivity Investigation

### Initial Failure

Workload Cluster の Pod から
MinIO に接続確認を実施した。

```bash
curl http://minio.lab.local
```

結果:

```text
Could not resolve host
```

DNS 解決に失敗した。

---

### Investigation

IP アドレスを直接指定。

```bash
curl -I -H "Host: minio.lab.local" \
http://192.168.11.18
```

結果:

```text
Server: MinIO
```

ネットワーク疎通は正常であることを確認した。

---

### Resolution

CoreDNS ConfigMap の NodeHosts を更新。

```text
192.168.11.18 minio.lab.local
192.168.11.18 console.minio.lab.local
```

適用後:

```bash
kubectl -n kube-system rollout restart deployment/coredns
```

再確認:

```bash
curl http://minio.lab.local
```

正常応答を確認した。

## Lessons Learned

### MinIO は1つ

MinIO は Management Cluster 上に存在する。

```text
rancher-mgmt-02
└── MinIO
```

Workload Cluster 側には存在しない。

Runtime Job は
Management Cluster の MinIO を利用する。

---

### API と Console は別

API:

```text
minio.lab.local
```

Console:

```text
console.minio.lab.local
```

同じ MinIO インスタンスである。

---

### Longhorn は別課題

今回利用した MinIO は

```text
PVC
↓
local-path
```

で構成されている。

Longhorn 導入は
Storage HA の課題であり、
今回の Runtime Execution とは別である。
