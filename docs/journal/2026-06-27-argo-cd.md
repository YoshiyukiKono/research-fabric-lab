よし、これは **Argo CD App of Apps** で入れる構成です。

次の順番で進めるのがよいです。

## 1. Argo CDを入れる

```bash
export KUBECONFIG=~/.kube/workload-k3s.yaml

kubectl create namespace argocd

kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

待ちます。

```bash
kubectl -n argocd rollout status deploy/argocd-server
kubectl -n argocd get pods
```

## 2. root-appを適用

```bash
cd ~/research-fabric-lab
kubectl apply -f deploy/argocd/root-app.yaml
```

## 3. 同期状況確認

```bash
kubectl get applications -n argocd
kubectl get ns | grep research
kubectl get crd | grep -E 'experiment|agenttask'
```

## 4. Argo CD UIを見る場合

まずパスワード取得。

```bash
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath='{.data.password}' | base64 -d; echo
```

NodePort化するなら：

```bash
kubectl -n argocd patch svc argocd-server \
  -p '{"spec":{"type":"NodePort"}}'
```

確認：

```bash
kubectl -n argocd get svc argocd-server
```

アクセスは：

```text
https://10.110.1.212:<NodePort>
```

まずは `kubectl get applications -n argocd` まで見ましょう。

The CustomResourceDefinition "applicationsets.argoproj.io" is invalid: metadata.annotations: Too long: may not be more than 262144 bytes

これは **Argo CDの巨大CRDを `kubectl apply` した時の有名なエラー**です。`last-applied-configuration` annotation が大きすぎます。

対処は、`apply` ではなく **server-side apply** を使います。

```bash
kubectl apply --server-side -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

もし途中まで入っているなら、一度 namespace ごと消してやり直すのが早いです。

```bash
kubectl delete namespace argocd
kubectl create namespace argocd

kubectl apply --server-side -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

その後：

```bash
kubectl -n argocd get pods
kubectl -n argocd rollout status deploy/argocd-server
```

今回のポイントはこれです。

```text
Argo CD CRDは大きい
kubectl apply は巨大annotationを付ける
annotation上限に引っかかる
--server-side なら回避できる
```


Argo CD本体は入りました。`applications` が空なのは正常です。まだ **root-app** を入れていません。

次はこれです。

```bash
cd ~/research-fabric-lab
kubectl apply -f deploy/argocd/root-app.yaml
```

確認：

```bash
kubectl get applications -n argocd
```

少し待ってから：

```bash
kubectl get applications -n argocd
kubectl get ns | grep research
kubectl get crd | grep -E 'experiment|agenttask'
```

もし同期が進まない場合は：

```bash
kubectl describe application root-app -n argocd
```

で原因を見ます。


原因はこれです。

```text
deploy/k8s/platform の kustomization が
Deployment argocd-server に patch を当てようとしているが、
その kustomization の resources 内に argocd-server Deployment が存在しない
```

つまり `research-platform` は **Argo CD本体をインストールするものではなく、既存のArgo CDに追加設定を当てる想定**なのに、Kustomize単体としては patch 対象が見つからず失敗しています。

今回すでに Argo CD は手動で入っているので、まずは `research-platform` は無効化でよいです。

```bash
kubectl delete application research-platform -n argocd
```

確認：

```bash
kubectl get applications -n argocd
```

これで全部 `Synced / Healthy` になればOKです。

その後、必要なら `argocd-server` の insecure patch は手動で当てます。

```bash
kubectl -n argocd patch deployment argocd-server \
  --type='json' \
  -p='[{"op":"add","path":"/spec/template/spec/containers/0/args/-","value":"--insecure"}]'
```

まずは `research-platform` を削除して進めるのが安全です。
