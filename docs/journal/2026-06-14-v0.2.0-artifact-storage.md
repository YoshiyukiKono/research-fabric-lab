# Journal: v0.2.0 Artifact Storage

日付: 2026-06-14

## 概要

v0.2.0 では Research Fabric Lab に Artifact Storage を導入した。

本リリースの目的は、実験実行結果を Kubernetes Job の中だけに留めず、永続的に保存できるようにすることである。

完成したパイプラインは以下の通り。

## 実施内容

### Storage Layer追加

以下をGitOps管理下で導入した。

* Namespace: storage
* MinIO
* Traefik Ingress
* PVC

### MinIO Console公開

以下のURLでアクセス可能となった。

* minio.lab.local
* console.minio.lab.local

ローカル環境では hosts への登録が必要。

### Runtime Upload Example追加

サンプルJobを実行し、MinIOへのアップロードを検証した。

## 発生した問題

### Longhorn未導入

当初は以下の設定としていた。

```yaml
storageClass: longhorn
```

しかし Management Cluster には Longhorn を導入していなかった。

そのため、

* PVC Pending
* Pod Pending

となった。

原因確認:

```bash
kubectl describe pvc -n storage minio
```

結果:

```text
storageclass.storage.k8s.io "longhorn" not found
```

対応:

```yaml
storageClass: local-path
```

へ変更。

### ArgoCD UI障害

MinIO導入作業中に ArgoCD UI が利用できなくなった。

ブラウザ上では CORS 関連エラーが表示された。

原因:

TraefikでTLS終端している一方で、argocd-server がHTTPS前提で動作していた。

対応:

argocd-server に以下を追加。

```text
--insecure
```

さらに GitOps 管理へ組み込み、再発防止を行った。

### App of Apps構成の理解

今回の作業で以下を確認できた。

research-fabric-root

↓

research-storage

↓

research-platform-minio

↓

Helm Chart

親Applicationを更新しないと子Applicationへ反映されないことを実体験として理解した。
