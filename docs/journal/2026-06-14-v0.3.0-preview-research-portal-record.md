# Journal: v0.3.0-preview Research Portal

日付: 2026-06-14

## 概要

v0.3.0-preview では、Workload Cluster である `agent-lab-02` 上に Streamlit ベースの Research Portal を配置した。

この段階では OpenAI API 呼び出しや Experiment Operator 連携は行わず、まず Workload Cluster 上に研究基盤の入口となる Web UI を表示できることを確認した。

## 構成

```text
agent-lab-02

namespace: research

research-portal
├─ ConfigMap
├─ Deployment
├─ Service
└─ Ingress
```

## URL

```text
http://portal.agent.lab.local
```

## hosts設定

Workload Cluster 側の Traefik Ingress は以下のIPで公開された。

```text
192.168.11.16
```

そのため、クライアント側の hosts に以下を追加した。

```text
192.168.11.16 portal.agent.lab.local
```

## 検証結果

Namespace 作成:

```text
namespace/research created
```

Pod 状態:

```text
research-portal Running
```

Ingress:

```text
portal.agent.lab.local 192.168.11.16
```

HTTP確認:

```text
HTTP/1.1 200 OK
```

## 表示内容

Research Portal では以下を表示した。

* Management Cluster: rancher-mgmt-02
* Workload Cluster: agent-lab-02
* Artifact Storage: MinIO
* Experiment Draft
* Runtime Image
* Command
* Artifact Path
* Generated Experiment Spec Preview

## 意義

v0.1.0 では GitOps 管理基盤を構築した。

v0.2.0 では MinIO による Artifact Storage を導入した。

v0.3.0-preview では Workload Cluster 上に Research Portal を配置し、研究者がアクセスする入口を作成した。

これにより、Research Fabric Lab は以下の構成を持つようになった。

```text
Management Cluster
  Rancher / ArgoCD / MinIO

Workload Cluster
  Research Portal / Runtime Jobs
```

## 学び

今回の作業では、Management Cluster 側と Workload Cluster 側で Ingress IP が異なることを確認した。

当初 `portal.agent.lab.local` が Management Cluster 側の `192.168.11.18` を向いていたため 404 となった。

正しくは Workload Cluster 側の Traefik Ingress IP である `192.168.11.16` を指定する必要があった。
