# Journal: v0.3.0-preview OpenAI Integration

日付: 2026-06-14

## 概要

Research Portal に OpenAI API 連携を追加する。

この段階では、Experiment Operator や GitHub Pull Request 生成までは行わず、まず Research Goal から Experiment Spec の案を生成し、Streamlit 上に表示する。

## 目的

```text
Research Goal
↓
OpenAI API
↓
Experiment Spec
↓
Streamlit Preview
```

## Secret管理

OpenAI API Key は GitHub に保存しない。

Workload Cluster の `research` namespace に Kubernetes Secret として作成する。

```bash
kubectl --context agent-lab -n research create secret generic openai-api-key \
  --from-literal=OPENAI_API_KEY='sk-...'
```

確認:

```bash
kubectl --context agent-lab get secret -n research
kubectl --context agent-lab describe secret openai-api-key -n research
```

## Deployment変更

`research-portal` Deployment は `OPENAI_API_KEY` を Secret から環境変数として参照する。

```yaml
env:
  - name: OPENAI_API_KEY
    valueFrom:
      secretKeyRef:
        name: openai-api-key
        key: OPENAI_API_KEY
        optional: true
```

## Portal変更

Streamlit UI に以下を追加する。

- Research Goal 入力欄
- Generate Experiment Spec with OpenAI ボタン
- AI生成結果のJSON Preview
- Manual Draftとの比較タブ

## 今後

次の段階では、生成された Experiment Spec を以下へ接続する。

```text
Experiment Spec
↓
GitHub Pull Request
↓
ArgoCD
↓
Experiment Operator
↓
Runtime Job
↓
MinIO Artifact
```
