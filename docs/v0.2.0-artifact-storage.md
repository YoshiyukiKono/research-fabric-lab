# v0.2.0 Artifact Storage

## Summary

v0.2.0 introduces artifact storage into Research Fabric Lab.

The main path is:

```text
Experiment
  -> Runtime Job
  -> result.json
  -> MinIO
  -> research-artifacts bucket
```

## Design decisions

### Namespace

MinIO is deployed into:

```text
storage
```

This keeps storage infrastructure separate from:

```text
research-system
research-experiments
argocd
```

### Object storage

MinIO is used because it gives the lab an S3-compatible artifact layer.

This is useful for later integrations with:

- Experiment Operator
- Research Portal
- MLflow
- JupyterHub
- AI Generated PR workflows
- HPC and quantum simulation workloads

### Persistent volume

MinIO uses Longhorn-backed PVC.

Initial size:

```text
100Gi
```

### Ingress

Two hostnames are assumed:

```text
minio.lab.local
console.minio.lab.local
```

### Bucket

Initial bucket:

```text
research-artifacts
```

## Validation

After ArgoCD sync:

```bash
kubectl get pods -n storage
kubectl get pvc -n storage
kubectl get ingress -n storage
```

Then run the sample upload job:

```bash
kubectl apply -k research-platform/storage/minio/runtime-upload-example
kubectl logs -n research-experiments job/artifact-upload-example
```

Expected output:

```text
uploaded: s3://research-artifacts/experiment-001/result.json
```
